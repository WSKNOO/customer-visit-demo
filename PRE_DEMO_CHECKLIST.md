# 演示前检查清单

## 凭据与权限

- [ ] 已完成 `CREDENTIAL_ROTATION_CHECKLIST.md` 中必须吊销的项目。
- [ ] 原始 Git 仓库已完成历史凭据扫描。
- [ ] 服务器 `.env` 权限为仅运行账号可读。
- [ ] 页面源码、浏览器请求和日志中没有完整 Key、Token、密码或内网模型地址。
- [ ] 统一网关访问控制、IP 白名单和限流已启用。

## 模型

- [ ] 已确认演示使用 Mock 还是受控真实模型。
- [ ] 真实模式使用的是新凭据、测试配额和非敏感演示客户。
- [ ] 内部模型域名已加入 `NO_PROXY`。
- [ ] 401、429、5xx 和超时提示已验证。
- [ ] AI陪练模型失败时可立即恢复 `TRAINING_MOCK_MODE=true`。

## 搜索与代理

- [ ] 公网代理地址、DNS和证书链已由运维确认。
- [ ] 搜索服务健康地址可达且不会产生付费业务调用。
- [ ] 客户官网、政府公开网站和新闻来源各验证一条。
- [ ] 抓取失败不会使情报 API 退出。
- [ ] `SEARCH_MAX_FETCH_PAGES`、`SEARCH_MAX_CONTENT_CHARS` 和超时已设上限。

## 服务健康

- [ ] 统一入口正常。
- [ ] Nginx `/gateway-health` 正常。
- [ ] 情报前端正常。
- [ ] 情报后端 `/intelligence-api/health` 正常。
- [ ] AI陪练前端正常。
- [ ] AI陪练后端 `/api/health` 正常。
- [ ] `deploy/pre-demo-healthcheck.sh` 全部必检项通过。
- [ ] AI陪练仅运行一个 Gunicorn worker。

## 页面与链路

- [ ] 统一入口三个按钮文案和链接正确。
- [ ] 方案助手正式 URL 已配置并可访问。
- [ ] 情报报告详情显示“开始AI陪练”。
- [ ] 创建会话时按钮有加载状态，失败时报告不会消失。
- [ ] 跳转 URL 只包含 `session_id`，不含完整情报或 Base64。
- [ ] 陪练页面自动显示客户、角色、目标和开场问题。
- [ ] 麦克风、ASR、TTS 默认关闭且不弹权限窗口。
- [ ] 三轮文字问答、评分和最终总结完整。

## 缓存与兜底

- [ ] 星海智造缓存结果可以加载。
- [ ] 云岭交通缓存结果可以加载。
- [ ] 缓存结果有醒目标识，不被表述为实时事实。
- [ ] 陪练 Mock 回答可用。
- [ ] 服务重启后能从报告重新创建会话。
- [ ] 已准备统一入口、情报、陪练、评分四张截图。
- [ ] 已准备备用录屏，并在演示电脑上离线播放验证。

## 演示现场

- [ ] 使用无痕窗口或已清理历史的浏览器。
- [ ] 关闭浏览器语音权限和无关通知。
- [ ] 演示电脑电源、网络、投屏和分辨率正常。
- [ ] 操作人已按 `DEMO_RUNBOOK.md` 完整走查一次。
