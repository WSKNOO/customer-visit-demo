## 1. 项目概述

### 1.1 系统定位
AI客户拜访模拟对练系统，基于大语言模型模拟真实客户角色，帮助售前工程师进行实战对话训练。系统提供实时评分、知识库辅助、语音交互等完整训练流程。

### 1.2 核心功能
| 功能模块 | 说明 |
|----------|------|
| **场景选择** | 3大分类 (AI/标品/解决方案)，60+ 细分场景，层级下钻 |
| **难度分档** | 简单(5轮)/中等(8轮)/困难(10轮)，客户行为不同 |
| **AI客户模拟** | DeepSeek V3 扮演真实客户，3轮后检测结束语 |
| **实时评分** | 5维度评分 (专业度/沟通/需求洞察/异议处理/成交引导) |
| **开挂模式** | RAG检索知识库，实时提示标准答复 |
| **语音交互** | 轻量模式靠浏览器 API，完整模式用 FunASR |
| **TTS播报** | Edge TTS 语音合成，AI回复自动朗读 |
| **评估报告** | 对话结束后弹窗显示评分+雷达图+改进建议 |

## 5. 核心模块详解

### 5.1 app.py 启动流程

```
main()
  ├── 解析命令行参数 (--full)
  ├── 初始化 RAG 检索器
  ├── (完整模式) 加载 FunASR 模型
  ├── 初始化知识库场景列表
  └── app.run('127.0.0.1', 5000)
```

### 5.2 双模式运行

```
轻量模式: python app.py
  ├── AI对话: DeepSeek V3 ✓
  ├── 语音识别: Web Speech API (浏览器)
  ├── 知识库: RAG按需加载 ✓
  └── 无需 torch/funasr ✓

完整模式: python app.py --full
  ├── 以上所有 + FunASR后端语音识别
  └── 需要: torch + funasr + modelscope
```

### 5.3 AI对话流 (`call_deepseek`)

```
1. RAG检索 (用最近对话搜索知识库)
2. 构建 System Prompt (场景+难度+知识库+评分规则)
3. 调用 DeepSeek V3 API
   ├── temperature: 0.8
   ├── max_tokens: 400
   ├── messages: 最近12条
   └── 3次重试 (90s超时)
4. 返回客户回复 + SCORE + COACH
```

### 5.4 开挂模式 (`/api/cheat`)

```
1. RAG双路检索
   ├── 完整对话上下文搜索
   └── 最后一条客户消息精准搜索
2. 排序去重取 Top-10
3. 调用 DeepSeek V3 生成
   ├── temperature: 0.3
   ├── max_tokens: 600
   └── frequency_penalty: 0.2
4. 返回: 诉求分析 + 标准答复 + 文档来源
```

## 6. 知识库与RAG检索系统

### 6.1 SimpleRAG 架构

```python
class SimpleRAG:
    chunks = []          # 检索块列表
    loaded_scenes = set() # 已加载场景缓存

    # 核心方法
    load_scene(scene_id)      # 加载场景 (卡片+文档)
    search(query, top_k=8)   # 检索
    format_context(results)   # 格式化输出
```

### 6.2 检索流程

```
用户提问
  ↓
分词 (标点切分 + 2字滑动窗口 + 停用词过滤)
  ↓
三维度打分:
  ├── 关键词命中 (50%) — 核心词匹配
  ├── 短语连续匹配 (35%) — 2-4字精确命中
  └── Jaccard字符相似 (15%) — 兜底
  ↓
排序 → 去重 → Top-10 → 格式化
```

### 6.3 数据源结构

| 类型 | 格式 | 来源 |
|------|------|------|
| **产品卡** | JSON (product/case/objection/solution/parameter) | `knowledge_cards.json` |
| **场景卡** | JSON (客户画像+痛点+异议+评分) | `scene_card.json` |
| **原始文档** | docx/pdf/pptx/xlsx/txt (自动分块) | knowcard子目录 |
| **检索块** | 每块 ≤400字, 含来源标注 | RAG内部缓存 |

### 7.2 状态管理

```javascript
state = {
    selectedScene: '',       // 当前场景ID
    selectedDifficulty: '',  // 难度
    messages: [],            // 对话历史
    msgCount: 0,             // 轮次计数
    roundLimit: 8,           // 轮次上限
    sessionStarted: false,   // 是否已开始
    conversationEnded: false,// 是否已结束
    ttsEnabled: true,        // TTS开关
```

### 7.3 关键交互

| 交互 | 实现 |
|------|------|
| 场景选择 | 一级分类 → 二级场景下钻，选中后调 `/api/scene/load` 预加载RAG |
| 发送消息 | Enter键 / 点击发送 / 语音输入 (Web Speech API) |
| 对话结束 | AI客户说结束语 / 轮数满 → 禁用输入 → 弹出报告 |
| 开挂模式 | 右侧 toggle 开关，AI回复后自动查询知识库 |
| 报告弹窗 | 点击遮罩或✕关闭，`继续练习`重置状态 |

### 7.4 角色禁止防护

```
AI扮演客户时的硬约束:
  - 禁止说 "太好了""非常棒""你答对了"
  - 对方说"好的"→必须继续质疑
  - 每句话包含: 质疑/压价/竞品/顾虑 之一
  - temperature 0.8 (避免过于确定)
```

## 10. 配置与调优

### 10.1 模型配置

```python
# 主对话 (客户模拟)
DEEPSEEK_MODEL = 'deepseek-ai/DeepSeek-V3'
temperature = 0.8
max_tokens = 400
context_window = 12条消息

# 开挂模式 (知识库答复)
CHEAT_MODEL = 'deepseek-ai/DeepSeek-V3'
temperature = 0.3
max_tokens = 600
frequency_penalty = 0.2
```

### 10.2 性能优化

| 优化项 | 实施 |
|--------|------|
| System Prompt | 精简至 ~400 tokens (原2000+) |
| 上下文窗口 | 限制最近12条消息 |
| RAG懒加载 | 场景切换时才加载知识库 |
| 文档分块 | PDF最多30页，每块400字 |
| PyPDF2保护 | 异常捕获+警告抑制 |
| 前端体积 | 单文件55KB，无框架依赖 |

### 10.3 检索质量

| 维度 | 策略 |
|------|------|
| 分词 | 标点切分 + 2字滑动窗口 |
| 停用词 | 50个中文停用词过滤 |
| 去重 | 指纹去重 (前50字符) |
| 来源追溯 | 每个检索块标注文档路径 |

## 附录A: 对话评分体系

| 维度 | 权重 | 评分范围 |
|------|:----:|----------|
| professionalism | 专业度 | 0-100 |
| communication | 表达沟通 | 0-100 |
| needs_analysis | 需求洞察 | 0-100 |
| objection_handling | 异议处理 | 0-100 |
| closing | 成交引导 | 0-100 |

每轮AI回复中嵌入 `<!--SCORE{...}-->` JSON，前端解析实时更新评分面板。

## 附录B: 结束条件

| 条件 | 触发 |
|------|------|
| 轮数满 | msgCount >= roundLimit |
| 客户结束语 | 含"今天就到这里""先去忙了"等(3轮后) |
| 情绪愤怒 | mood=angry + 倒数第2轮 |
| 报告标记 | 回复含 `<!--REPORT` |

结束后: 输入框禁用 → 弹窗显示完整报告 → "继续练习"重置。
