## 四、核心模块详解

### 4.1 后端服务 (`app.py`)

#### 4.1.1 配置管理

```python
# 环境变量 (.env 文件加载)
DEEPSEEK_API_KEY    → 硅基流动 API Key
DEEPSEEK_BASE_URL   → https://api.siliconflow.cn/v1
DEEPSEEK_MODEL      → deepseek-ai/DeepSeek-V4-Pro
DOC_PATH            → ./doc (知识库路径)
TTS_VOICE           → zh-CN-YunxiNeural (默认男声:云希)
```

#### 4.1.2 API 接口清单

| 方法 | 路径 | 功能 | 参数 |
|------|------|------|------|
| `GET` | `/` | 返回前端页面 | - |
| `POST` | `/api/chat` | AI 对话核心接口 | messages[], difficulty, scene |
| `POST` | `/api/tts` | 文字转语音 (返回 MP3) | text, voice |
| `GET` | `/api/tts/voices` | 获取可用语音列表 | - |
| `POST` | `/api/asr` | 语音转文字 (完整模式) | audio(Base64), format |
| `GET` | `/api/knowledge` | 获取知识库信息 | - |
| `GET` | `/api/mode` | 获取运行模式 | - |
| `GET` | `/api/health` | 服务健康检查 | - |
| `POST` | `/api/test` | 测试 API 连通性 | - |

#### 4.1.3 AI 对话引擎 (`call_deepseek`)

**工作流程：**
```
用户输入 → 构建 System Prompt(含场景/难度/规则) → 调用 DeepSeek API → 返回回复
```

**System Prompt 核心能力：**
- 🎭 **真实客户角色扮演**：不引导回答，像真实客户一样提需求/质疑/压价
- 📊 **动态难度控制**：简单5轮 / 中等8轮 / 困难10轮
- ⚠️ **答非所问检测**：3次跑题 → 提前终止 → 直接进入复盘
- 📋 **五维评分体系**：需求挖掘(20) + 方案匹配(20) + 异议处理(20) + 话术表达(20) + 促成推进(20)

#### 4.1.4 TTS 引擎 (`text_to_speech`)

```
文本 → asyncio → edge_tts.Communicate → MP3 音频流 → HTTP Response
```

- 使用微软 Edge 在线神经 TTS，无需本地模型
- 默认男声：`zh-CN-YunxiNeural` (云希)
- 支持 8 种中文声音切换
- 异步生成 + 同步封装（Flask 兼容）

#### 4.2.3 核心功能模块

| 模块 | 功能 | 关键函数 |
|------|------|----------|
| **环境检测** | 自动判断 HTTP/HTTPS/localhost，提示麦克风限制 | `initEnvironment()` |
| **语音识别 (ASR)** | Web Speech API 实时转写，支持中英文混合 | `initSpeechRecognition()` |
| **对话管理** | 消息收发、历史记录、格式化渲染 | `callAI()`, `addMessage()` |

| **对练流程控制** | 场景选择、难度配置、轮次计数、自动结束 | `startNewSession()` |
| **导出功能** | 导出对话记录为 TXT 文件 | `exportChat()` |

### 4.3 HTTPS 服务 (`https_server.py`)

用于公网部署时解决浏览器麦克风权限问题：

```
启动流程:
1. 生成自签名 SSL 证书 (RSA-2048, 有效期10年)
2. 创建 SSL Context (TLS 1.2/1.3)
3. 后台线程监听 5000 端口 → 301 重定向到 443
4. Flask 以 ssl_context 启动在 443 端口
```

## 五、数据流图

### 5.1 完整对练流程

```
[用户] 选择场景 + 难度
    ↓ 点击「开始对练」
[前端] 发送 "@开始" + 场景/难度参数 → POST /api/chat
    ↓
[后端] 构建含规则的 System Prompt → 调用 DeepSeek API
    ↓
[DeepSeek] 返回客户开场白 (模拟真实客户)
    ↓
[前端] 显示 AI 消息 → 自动调用 TTS 播放语音
    ↓
[用户] 说话 (麦克风) 或 打字 (键盘)
    ↓
[前端 ASR] Web Speech API 实时转写 → 填入输入框 → 发送
    ↓
[后端] 接收用户消息 + 判断是否答非所问
    ├─ 正常 → 追加到上下文 → 调用 DeepSeek → 返回客户回应
    └─ 跑题3次 → 提前终止 → 直接进入复盘评分
    ↓
[循环直到达到轮数或提前终止]
    ↓
[后端] 切换教练身份 → 五维评分报告
    ↓
[前端] 显示评估报告 + TTS 朗读摘要
```

### 5.2 语音处理链路

```
【输入链路】
用户语音 → 浏览器麦克风采集 → Web Speech API (实时转写)
    → interimResults(临时结果) + final(最终确认)
    → 自动填入输入框 → 用户确认发送

【输出链路】
AI 回复文本 → POST /api/tts {text, voice}
    → edge_tts.Communicate (异步流式生成 MP3)
    → Response(audio/mpeg) → Blob → URL.createObjectURL
    → new Audio() → play()
```

## 七、运行与部署

```bash
# 1. 安装依赖
pip install -r requirements-lite.txt

# 2. 配置 .env 文件 (API Key 等)

# 3. 启动服务 (轻量模式)
```

### 7.2 云端部署

**服务器环境：** Ubuntu 22.04 / Python 3.8+

```bash
# 项目路径: /root/voice-practice/

# 服务进程
cd /root/voice-practice && nohup python3 https_server.py > https.log 2>&1 &

# 访问地址

HTTP:  http://${DEPLOY_HOST}:5000 (自动跳转)

```

## 九、AI Prompt 设计要点

### 9.1 角色设定

- **不是考官** — 不按清单提问
- **不是产品说明书** — 不主动引导回答
- **是真实客户** — 只关心痛点/价格/风险/竞品

### 9.2 难度梯度

| 难度 | 轮数 | 客户画像 | 特征 |
|------|:----:|---------|------|
| 简单 | 5 | 和蔼采购经理 | 配合度高、异议少 |
| 中等 | 8 | 谨慎决策者 | 2-3个异议点、比较供应商 |
| 困难 | 10 | 挑剔大老板 | 强势压价、竞品施压、时间紧迫 |

### 9.3 评分维度

| 维度 | 分值 | 评估标准 |
|------|:----:|---------|
| 需求挖掘 | /20 | 主动了解客户痛点 vs 被动回答 |
| 方案匹配 | /20 | 针对具体场景 vs 泛泛而谈 |
| 异议处理 | /20 | 有理有据化解 vs 逃避或硬推 |
| 话术表达 | /20 | 口语化简洁有力 vs 官腔书面语 |
| 促成推进 | /20 | 引导下一步行动 vs 无疾而终 |
