提供语音导航流程实现自动语音应答，可支持定制不同的多层级IVR导航流程，也可支持分时段导航，满足不同的业务咨询需要。
ACD话务分配
统一号码呼入、呼出，通过智能路由进行话务的分配、分机转接等话务接续处理。
统一号码呼入、呼出
在传统的电话呼叫系统中，企业或政府部门通常会使用多个电话号码来分别处理不同的服务领域，例如客户服务、投诉、咨询等。而“统一号码呼入、呼出”则意味着所有的呼入和呼出电话都通过一个唯一的电话号码来处理，简化了用户的拨打过程并提高了系统的管理效率。这个统一号码不仅是来电接入的通道，也可能用于系统主动发起的呼叫。
呼入：用户通过统一号码拨打电话，接入呼叫中心。无论用户的需求是咨询、投诉还是办理政务事务，所有电话都通过这个统一号码进行接入。
呼出：系统或客服人员根据需要主动拨打电话给用户。例如，系统主动发出通知、办理进度查询等，或者人工客服为了解决问题主动回电。
智能路由
智能路由是指通过算法和规则，对来电进行智能判断并将其引导至合适的处理环节或人员。智能路由系统基于一定的规则、优先级、业务领域等因素，自动将用户的电话分配给合适的处理单元。智能路由的目标是优化资源利用，减少用户等待时间，并确保用户能够快速获得与其需求匹配的服务。
1、智能路由的实现方式：
IVR（交互式语音响应）：系统首先通过IVR语音提示，引导用户选择服务类型（例如“按1查询税务，按2办理社保…”）。系统根据用户选择的服务类型进行路由，将来电引导到对应的队列或分机。
语音识别：如果不使用传统的按键选择，系统可以通过语音识别技术来理解用户的需求。例如，用户说出“我要办社保”，系统可以直接识别并将来电路由到社保服务部门。
人工智能（AI）支持：一些更复杂的智能路由系统可能会利用AI分析来电的内容，判断用户的情绪、问题类型等，以此来调整路由规则。AI可以根据历史数据和用户行为预测需求，自动选择最合适的处理路径。
2、路由规则：
服务领域分类：根据用户的需求（例如，咨询、投诉、办理业务），系统可以将来电转接到相应的业务部门或专业客服人员。
话务量和优先级：在某些情况下，系统会根据话务量的高低、处理的紧急性（例如，投诉或紧急事务）来调整路由优先级。
技能路由：智能路由可以根据客服人员的技能和专长，将电话分配给具备相关知识和能力的人员。例如，某些客服人员可能专门处理税务问题，而另一些则专注于社会保障领域。
话务的分配
在统一号码呼入的情况下，话务的分配指的是如何将一个来电分配给适当的客服人员或服务团队。这一过程不仅仅是将电话接通，而是要确保每个电话能得到高效处理并最大化利用资源。
基于队列的分配：所有的来电按照先后顺序进入不同的服务队列，系统按照队列顺序依次分配电话给客服人员。例如，如果“税务”队列的客服空闲，系统会将来电分配给这个队列的工作人员。
基于技能和专业的分配：一些复杂的系统会根据员工的技能和经验进行话务分配。如果某个问题需要专业知识，系统会确保将电话转接到该领域的专家。
动态负载均衡：智能路由系统可能会结合多个因素（如话务量、客服可用性、服务时效等）动态地调整话务分配策略，确保高峰时段的处理效率。
分机转接
分机转接是指在电话接入后，根据用户需求或问题的复杂性，将电话从一个接入点转接到另一个分机或部门。例如，初级的客户服务人员可能只能处理常规问题，对于需要进一步处理的复杂问题，电话会被转接到高级技术支持或专门的部门。
分机转接的过程：
人工接听：用户的电话首先由接听人员接听，接听人员根据用户的需求决定是否需要转接。
转接到专业团队：如果问题涉及到更专业的领域，接听人员会将电话转接到相关部门或专业人员。例如，税务问题转接到税务部门，社保问题转接到社会保障部门。
按服务时段转接：如果某个部门在特定时段不办公，系统可以根据时间自动转接到其他可用部门。
自动转接与人工转接：系统可以自动识别一些简单的问题并给予解答，如果是较为复杂或需要人工服务的问题，则由人工客服转接处理。
录音系统
提供电话录音功能，实现通话呼入后，系统进行电话录音，用于热线进行调听并管理，保证服务质量，可提供录音文件进行查询、调听、转存等相关功能。
坐席监控
实时监控坐席的登录状态、忙碌状态、小休状态等，确保坐席资源的合理分配和利用。可以实时监控热线的工作状态，包括呼叫量、平均处理时间等关键指标。
软电话控件
为服务人员提供系统便捷的电话通信功能，能够进行高效便捷的沟通，包括电话接听、外拨、通话保持、挂机、转三方、签入、签出、满意度评价等操作功能。
1、电话接听
电话接听功能是服务人员最基本的功能，确保服务人员能够顺利接入用户来电，开始为用户提供服务。
2、通话保持
通话保持功能允许服务人员在与客户通话过程中，将电话暂时保持，进行一些后台操作或与其他人员协调。
3、挂机
挂机功能是服务人员结束通话的基本操作。在通话结束后，服务人员可以通过挂断电话按钮完成通话，并开始处理后续操作。
4、转三方
转三方功能（也叫会议呼叫或三方通话）使得服务人员能够将当前通话转接到其他人员或部门，以便进行更高效的沟通和问题解决。
5、签入与签出
签入和签出功能主要用于服务人员的工作时间管理和任务分配。这些功能确保服务人员按时开始和结束工作，并能准确记录其工作状态。
智能语音客服
红黑名单管理
红名单电话功能： 也叫VIP名单，该号码具有优先接入或接入vip专席，由vip坐席对其进行服务

坐席工号管理
坐席工号管理，核心目标在于实现对坐席工号的高效有序管理，保障坐席工号能得到科学合理的分配与运用。此系统具备全面且实用的功能，
查询功能：方便快速定位所需工号信息；
新增功能：可灵活应对因业务拓展产生的新工号需求；
修改功能：能及时调整工号相关的各类信息；
删除功能：妥善处理不再使用的工号，避免数据冗余。
通过这些功能，系统全力确保坐席工号数据始终维持在准确无误且实时更新的状态，为整个业务流程的顺畅运转提供坚实的数据支撑。
技能组管理
技能组管理系统的核心使命在于实现技能组的高效管理，致力于保障技能组能够得到合理的分配与充分运用。
在功能层面，该系统提供了一系列关键操作功能：
查询功能：使用户能够便捷、迅速地从系统中获取特定技能组的相关信息，极大地提升信息获取效率。
新增功能：支持用户根据实际业务发展与需求，及时在系统内创建新的技能组，确保系统能够紧密贴合不断变化的业务场景。
修改功能：当技能组的属性、成员构成等信息发生变化时，用户可通过此功能对技能组数据进行精准调整，保障数据与现实情况一致。
删除功能：针对不再具有使用价值或已过时的技能组，用户能够在系统中进行清理操作，维护系统数据的简洁性与有效性。

平台概述
具备统一的门户服务应用、话务调度系统、工单办理系统、热线知识库系统、数据辅政系统、综合管理系统等应用。
平台提供7×24小时全天候电话接听服务，同时依托政务服务网、微信公众号和小程序等各类互联网渠道获取群众、企业诉求，实现服务对象诉求的受理、审核（派发）、办理、协调、督办、办结、回访、评价等闭环服务，支持服务对象通过诉求提交原渠道及时跟进了解诉求办理进度、诉求反馈结果。
业务场景
智能语音客服
智能语音客服可提供与社会公众的智能交互，直接受理市民简单高频的问题诉求，包含业务直达、二次/多次引导、转接人工坐席等功能，辅助人工坐席提供自主在线问答、咨询、业务办理等服务。
智能文本客服
基于自然语言处理和多轮会话方式，统一服务来自网站、APP、微信、H5等各渠道客户诉求，系统支持文字、图片、语音、表情、文件等多种新媒体消息，通过识别用户语义，对咨询类问题直接推送标准答案，无法解答的问题进行转人工服务。
智能坐席助手（智能填单）
智能填单可以将诉求人和话务代表的语音对话分别实时转化为文本，同时可自动写入来电号码、识别问题描述等关键信息，自动生成工单内容摘要和概括性标题，供话务员复核后决定是否采用。支持一键复制填入工单系统字段项和人工修改输入。
智能坐席助手（话术推荐）
与知识库系统结合，为坐席提供知识推荐跟随、话术指引。在话务指服务中，根据当前用户所反馈的事项，提供关键要素描述，并根据当前语音沟通，对关键要素是否获取进行标识，根据用户描述的内容，结合上下文信息，向坐席推荐相关知识。
智能派单
基于历史派发数据进行深度学习，构建智能分派算法模型，从而自动推荐可派单的成员单位，在派单过程中，通过不断学习积累分派数据，实现分派准确率的不断优化提升。
智能知识库
提供知识新增、审核、上架、发布、下架等全生命周期的统一知识服务，它还运用语义理解技术提供更强大的检索能力，提升支持在话务员在线解答、办理员查找答复依据能力，推动诉求高效办结。
智能报告
自动化报告基于大模型智能分析能力，将图文组件、数据标签、专题场景、报表样本等进行零件化组合，通过模板配置可实现报表的快速上线，为管理者量身打造分析研判的智能报告。

解压数据库文件夹 
tar -xzvf /data/software/mysql-5.7.17-linux-glibc2.5-x86_64.tar.gz
移动数据库压缩文件到指定目录下，并改名为mysql
mv mysql-5.7.24-linux-glibc2.12-x86_64/ /usr/local/mysql
新建数据库组
groupadd mysql
创建数据仓库
mkdir /data/mysql
新建mysql用户禁止shell访问
useradd -r -s /sbin/nologin -g mysql mysql -d /usr/local/mysql
给数据库安装地址只允许mysql用户访问，数据文件赋权755
chown -R mysql:mysql /usr/local/mysql
chmod -R 755 /usr/local/mysql
chmod -R 777 /usr/local/mysql
chown -R mysql:mysql /data/mysql
yum install -y libaio
在数据库bin目录下运行mysqld文件编译
./mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/data/mysql
(mysql8)./mysqld --initialize --user=mysql --basedir=/usr/local/mysql --datadir=/data/mysql --lower_case_table_names=1
*编译后会出现数据库登录账号密码，需要记住数据库登录账号密码！！！！一定要保存！！！
复制数据库配置文件到/etc/下，并改名my.cnf
cd support-files
cp my-default.cnf /etc/my.cnf
复制数据库启动文件到init.d目录下，并改名为mysql
cp mysql.server /etc/init.d/mysql
编辑数据库启动文件启动，并修改如图内容
vim /etc/init.d/mysql
basedir=/us
