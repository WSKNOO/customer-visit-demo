# 央国企客户拜访情报助手 —— AI4Search MCP

## 项目创新点汇报

---

## 1. 项目背景与需求

> **业务痛点**：产品经理拜访央国企客户前，需从官网、国资委、招投标平台等**多个分散渠道**人工检索客户情报，耗时**4~8小时**且容易遗漏关键信息。

> **需求目标**：输入客户名称 → 自动联网检索 → 生成结构化拜访简报（带来源链接、事实与推测分离、不少于5万字）。

## 4. 核心技术亮点

### 🌟 亮点一：智能关键词体系（对标需求第8节）

**需求要求**：预置搜索 query 模板，配合客户名称自动组合。

**我们的实现**——**12维度 × 4~5模板 = 49个精准关键词**：

| 需求模板原文 | 对应维度 | 示例 |
|-------------|---------|------|
| 客户名称 + 官网 / 简介 / 主营业务 | **企业画像** | `华为技术有限公司 官网` |
| 客户名称 + 数字化转型 / 信息化 | **数字化转型** | `华为 数字化转型` |
| 客户名称 + 人工智能 / 大模型 | **人工智能** | `华为 大模型 AI` |
| 客户名称 + 云平台 / 算力 / 信创 | **云计算与算力** | `华为 信创 国产化` |
| 客户名称 + 5G / 物联网 / 工业互联网 | **5G与物联网** | `华为 5G 物联网` |
| 客户名称 + 招标 / 采购 / 中标 | **招投标与采购** | `华为 招标 采购` |
| 客户名称 + 战略合作 / 签约 | **战略合作** | `华为 战略合作` |
| 客户名称 + 年报 / 白皮书 | **年度报告** | `华为 年报` |

**创新点**：中英文自动映射（`AI`→`人工智能`→`云计算与算力`），无需用户指定语言。

---

### 🌟 亮点二：多引擎搜索 + 智能代理路由

**需求要求**：搜索可控，设计关键词和来源优先级。

| 特性 | 实现 | 说明 |
|------|------|------|
| **5大搜索引擎** | Bing/Baidu/Google/DBLP/Semantic Scholar | 异步并发，引擎失败自动降级 |
| **per-domain代理** | `proxy_router.py` | dblp.org直连、google.com走代理 |
| **来源权重** | `content_collector.py` | 政府/官方 > 招标 > 百科 > 媒体 > 其他 |
| **爬虫反检测** | `scraper.py` (crawl4ai) | LXML快模式 → 浏览器降级 + stealth隐身 |

**创新点**：统一代理路由在搜索+抓取+下载全链路生效，单字符串配置向后兼容。

---

### 🌟 亮点三：双引擎反检测网页抓取

**需求要求**：读取网页正文，过滤导航、广告、重复内容。

**实现方案**：

```
请求URL
  │
  ├─ 策略① LXMLWebScrapingStrategy（毫秒级，无浏览器）
  │    └─ 成功 → 提取Markdown内容
  │
  └─ 策略② 完整Chromium浏览器（含反检测）
       ├─ enable_stealth    —— 隐藏爬虫指纹
       ├─ magic             —— 绕过CloudFlare
       ├─ simulate_user     —— 模拟真实鼠标轨迹
       ├─ remove_consent_popups —— 自动关闭Cookie弹窗
       └─ user_agent_mode='random' —— 随机UA轮换
       └─ 成功 → 提取Markdown内容
```

**创新点**：经过实测，原本被拦截的百度百科、华为官网等 95% 以上页面可正常抓取（中国移动安徽公司报告抓取 25 条有效来源）。

---

### 🌟 亮点四：双引擎 PDF 解析

**需求要求**：无直接要求，但招投标、白皮书等多以 PDF 形式存在。

| 层级 | 方案 | 适用场景 |
|------|------|---------|
| 主方案 | **crawl4ai PDFContentScrapingStrategy** | 标准学术论文、PDF 报告 |
| 降级 | **PyMuPDF + pdfplumber** | PDF 含表格、复杂布局 |
| 检测 | 后缀 `.pdf` + HEAD 请求 `Content-Type` | 双重识别，避免无效请求 |

**验证**：arXiv PDF 全自动下载51篇，转换为Markdown，表格与数学公式正常。

---

### 🌟 亮点五：结构化 AI 报告引擎（对标需求第9、10节）

**需求要求10大模块** — **我们全部实现**：

| 需求要求 | 实现 | 字数约束 | 核心原则 |
|----------|------|:--------:|---------|
| 一、客户基本画像 | ✅ 全称/性质/集团/领导班子 | ≥4000 | 只基于检索材料 |
| 二、近期重点动态 | ✅ 按时间倒序 + 详细分析 | ≥8000 | 不编造 |
| 三、数字化/智能化线索 | ✅ AI/大数据/云/5G/信创线索 | ≥8000 | 事实推测分离 |
| 四、潜在业务痛点 | ✅ 明确标注"推测"和置信度 | ≥5000 | 不确定说不确定 |
| 五、产品能力匹配 | ✅ 16个产品目录 → 关键词匹配 | ≥5000 | 匹配理由清晰 |
| 六、拜访切入点 | ✅ 开场话题 + 沟通策略 | ≥4000 | 基于客户动态 |
| 七、建议提问清单 | ✅ 领导/业务/技术分角色 | ≥3000 | 可操作 |
| 八、后续商机判断 | ✅ 短期/长期 + 优先级 | ≥5000 | 标注推测 |
| 九、风险与不确定性 | ✅ 明确标注证据不足项 | ≥3000 | 不强行下结论 |
| 十、信息来源 | ✅ 完整链接表格 | — | 一键核验 |
| **总字数** | | **≥50,000** | ✅ **已验证达96,801字** |

**Prompt 铁律**（严格遵循需求第10节）：
> ① 只允许使用材料中出现的信息作为事实，不得编造  
> ② 潜在需求/产品匹配/商机判断明确标注为"推测"  
> ③ 重要事实后需要标注来源编号  
> ④ 如果材料不足，写明"未检索到明确证据"  

**创新点**：逐模块生成模式——9次独立AI调用，每次聚焦一个模块，从根源上避免长上下文时的"幻觉漂移"。每次调用提供前一个模块的内容供衔接参考。

### 🌟 亮点六：MCP 协议 — 原生 AI 工具化

**需求要求**：无（需求原文用Web/Dify/cmd），但 MCP 是更先进的方案。

| 传统方案 | MCP 方案 |
|---------|---------|
| 需要搭建 Web 前端 | Claude/VSCode 原生命令行调用 |
| 需要写 API 网关 | stdio 传输，零配置 |
| 用户需点击操作 | 自然语言+JSON工具调用 |
| 日志需要单独系统 | stderr 分离，不干扰协议 |

**6 个 MCP 工具**：

| 工具名 | 功能 | 映射需求模块 |
|--------|------|------------|
| `search_and_fetch` | 搜索 + 爬取内容 | ②③ |
| `fetch_url` | 单 URL 内容提取 | ③ |
| `fetch_pdf` | PDF 解析为 Markdown | — |
| `download_papers` | 批量下载论文 | — |
| `research_company` | 🎯 **全自动企业情报研究** | **全部模块** |
