# 周四演示运行手册

## 1. 演示入口

- 统一入口：由运维提供的 `https://<正式域名>/`
- 本地/Mock 默认：`http://127.0.0.1:8080/`
- 当前应用没有内置演示账号；如 AI 中心网关增加认证，使用运维单独提供的账号，不在本文件记录密码。

主演示坚持：**缓存情报 + 纯文字陪练**。实时搜索和真实模型只在演示前受控验证通过后作为可选能力。

## 2. 推荐演示客户

1. `星海智造演示客户`：制造业、数据协同、质量追溯。
2. `云岭交通演示客户`：交通治理、运行监测、应急协同。

两者均为虚构数据，页面明确标注“缓存演示结果”，不会产生真实客户事实混淆。

## 3. 演示前十分钟

1. 执行 `PRE_DEMO_CHECKLIST.md`。
2. 运行 `deploy/pre-demo-healthcheck.sh`。
3. 打开统一入口，确认三个卡片都有正确按钮。
4. 加载星海智造缓存结果并创建一次陪练会话。
5. 确认麦克风、ASR 和 TTS 均关闭。
6. 完成一轮文字输入后重新从情报报告开始，避免展示测试历史。
7. 准备备用录屏和四张关键截图：统一入口、情报报告、三轮陪练、最终评分。

## 4. 标准演示流程

### 第一步：统一入口

打开“政企客户智能拜访助手”，简要说明三个阶段：情报准备、AI陪练、方案查询。

预期：三个卡片完整显示；方案助手按钮来自环境配置。

### 第二步：进入情报助手

点击“进入情报助手”→“发起新研究”→“加载 星海智造演示客户”。

预期：自动进入报告详情，页面显示“当前展示Mock或缓存演示结果”。

讲解重点：客户概况、近期动态、数字化线索、潜在需求、产品方向和建议提问。

### 第三步：开始AI陪练

点击报告顶部“开始AI陪练”。

预期：按钮显示加载状态，随后跳转到陪练页面；页面自动出现客户名称、角色、拜访目标和开场问题。用户无需再次填写资料。

### 第四步：三轮纯文字回答

建议话术：

1. “我们先了解生产与质量数据现状，再共同确认最影响业务的指标。”
2. “建议先梳理数据源、质量口径和异常处理时长，用这些基线判断方案是否适合。”
3. “可以选择一个工厂和一种高频质量异常做小范围试点，以定位时长和闭环率作为成功指标。”

预期：每轮出现客户追问和五维评分；第三轮生成最终报告和改进建议。

### 第五步：方案助手

返回统一入口，点击“进入方案助手”。只展示外部已上线能力，不介绍本轮代码集成。

## 5. 故障切换

| 故障 | 现场动作 |
|---|---|
| 实时搜索失败或超时 | 立即加载星海智造或云岭交通缓存结果。 |
| 情报模型失败 | 保留当前页面，使用缓存演示客户；不要重复连续提交。 |
| 陪练初始化失败 | 页面会保留报告；等待数秒重试一次。仍失败则直接从统一入口进入通用陪练或播放备用录屏。 |
| 陪练模型失败 | 将 `TRAINING_MOCK_MODE=true` 并重启单服务，再从报告重新创建会话。 |
| 会话过期/服务重启 | 返回情报详情重新点击“开始AI陪练”。 |
| 评分标记异常 | 展示最近一轮分数，切换备用录屏中的最终报告。 |
| 方案助手不可达 | 说明外部独立系统网络异常，使用截图展示，不修改本系统。 |
| 页面白屏 | 刷新一次；仍失败则检查网关和对应健康接口，切换录屏。 |

## 6. 演示结束

- 不导出或传播含敏感客户内容的对话记录。
- 清理浏览器历史摘要或使用无痕窗口。
- 记录请求编号、异常时间和服务名，不截图密钥或 `.env`。
- 如果临时启用真实模式，演示结束后恢复 Mock，并核对费用和调用日志。
